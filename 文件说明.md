# Z-Image Trainer v2 - 文件说明

## 目录结构

```
v2/
├── start.bat / start.sh                # 启动脚本 (Windows / Linux)
├── setup.bat / setup.sh                # 首次部署脚本
├── env.example                         # 环境变量模板
├── requirements.txt                    # Python 依赖
├── config/                             # TOML 配置预设 (9 个)
│
├── backend/                            # 后端 (Python + FastAPI)
│   ├── trainer_core/                   # 训练核心
│   │   ├── zimage_trainer/             # ⚙️  核心训练包 (独立, 不依赖外部 src/)
│   │   ├── shared/                     # 🔧 共享组件 (所有模型通用)
│   │   │   ├── losses/                 # Loss 函数
│   │   │   ├── optimizers/             # 优化器 (AdamW, FP8, BF16, Prodigy)
│   │   │   ├── lr_schedulers/          # 学习率调度器
│   │   │   ├── dataset/                # 数据加载
│   │   │   ├── gradient.py             # 梯度裁剪
│   │   │   ├── memory.py               # 显存管理
│   │   │   └── snr.py                  # SNR 加权
│   │   └── models/                     # 🧩 模型包 (约定式)
│   │       ├── zimage/                 # Z-Image 模型实现
│   │       └── _template/              # 📋 模板
│   │
│   ├── domain/                         # DDD 领域层
│   │   ├── training/                   # 训练管理
│   │   ├── dataset/                    # 数据集
│   │   ├── generation/                 # 图片生成
│   │   └── system/                     # 系统监控
│   │
│   ├── application/                    # DDD 应用层 (用例编排)
│   ├── infrastructure/                 # DDD 基础设施层
│   └── interface/                      # DDD 接口层 (FastAPI)
│       ├── main.py                     # 入口 (uvicorn backend.interface.main:app)
│       └── ...                         # Routers + DTO + WebSocket
│
└── frontend/                           # 前端 (Vue 3 + Element Plus)
    └── src/
```

## 启动方式

```bash
# 首次部署
./setup.bat    # Windows
./setup.sh     # Linux

# 启动服务
./start.bat    # Windows (默认端口 28000)
./start.sh     # Linux
./start.sh --port 8080 --dev  # 自定义端口 + 开发模式
```

## 设计原则

- **完全独立**: v2 不依赖旧项目 `src/` 目录, `zimage_trainer` 包已内嵌
- **DDD 四层**: Domain → Application → Infrastructure → Interface
- **共享核心 + 约定式模型包**: `shared/` 通用, `models/` 按模型独立
- **扩展新模型**: 复制 `_template/` 目录实现 6 个文件
